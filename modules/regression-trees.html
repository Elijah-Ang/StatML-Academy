Regression Trees

A visual guide to predicting numbers through recursive logic, top-down optimization, and simple math.

Stage 01

The Raw Target

A regression tree is a model that helps us predict a number. Not a category like "cat or dog", but a continuous value like an exam score, temperature, or a house price.

Let's look at a small dataset of houses. We want to predict the Price (in thousands) based entirely on the Size of the house (in square meters).

Look at the canvas. The horizontal axis is Size ($m^2$), and the vertical axis is Price ($k). Hover over the canvas to see active tooltips.

Stage 02

The Clumsy Guess

At the very start, the tree puts all houses in one big group (the Root Node). Before asking any questions, it asks: "If I had to guess one price for all houses, what should I use?"

The standard answer is the average. If we take all 16 training house prices and average them, we get exactly $483.1k.

Root Prediction = Average(y) = 483.1

This is a clumsy guess. The red lines on the canvas show the Residuals—the mathematical distance between our flat guess and reality.

Stage 03

The First Split

Now the tree asks a question to divide the data: "Can I split the houses into 2 groups so the prices inside each group become more similar?"

It checks a "cut point". Based on the math, the best first rule is: Is Size < 100?

If Yes, go Left. If No, go Right.

Now, instead of one average for everything, we have two averages. A lower average ($363.3k) for smaller houses, and a higher average ($637.1k) for bigger houses.

Stage 04

The Error Engine (RSS)

How did it know to split exactly at 100? The algorithm is "Greedy". It sweeps across every single possible threshold and chooses the one that leaves the least amount of mess.

It measures "mess" using the Residual Sum of Squares (RSS). It squares all the red error lines and adds them up.

Stage 05

Recursive Splitting

After finding the best first split, the tree does the exact same thing inside each new group. It splits, then splits inside the split. Like cutting a cake into pieces, then cutting those pieces again.

This is called Recursive Binary Splitting. With every split, the prediction line stops looking flat and starts looking like a set of stairs.

Watch the regions split and slide to form deeper nodes.

Stage 06

The Anatomy of a Tree

Even though we call it a tree, data scientists draw them upside down.

  • Root Node: The top, where all data starts.
  • Internal Node: A fork in the road.
  • Terminal Node (Leaf): The final box that gives the prediction number.
Interactive: Move your mouse over the canvas right now. The tree diagram will light up to trace the exact decision path for the region your cursor is in.

Stage 09

Trees vs. Lines

Linear regression fits a smooth, straight line. It assumes a constant rate of change. Regression trees carve the world into rectangles and assign a flat average to each.

If the data has sudden thresholds or "if/then" behaviors, trees capture it naturally. If it's a smooth trend, lines win.

Stage 11

Synthesis

You now understand the mechanics of a regression tree.

  1. Predicts a number, not a class.
  2. Greedy search finds the best splits to minimize RSS.
  3. Leaves predict averages of the data inside them.
  4. Stopping rules prevent overfitting, prioritizing generalization over memorization.

While a single tree can be unstable, this exact recursive splitting is the fundamental building block for advanced modern algorithms like Random Forests and Gradient Boosting.
