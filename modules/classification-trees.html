Every problem starts with data. We have measurements for fruit — Weight (x-axis) and Color Intensity (y-axis).

Stage 02

The Root Node

At the very beginning, a Classification Tree puts all the data into one big group on the fruit map. We call this the Root Node.

This starting box is very "messy" because it contains a mix of both apples and oranges. The tree's job is to split this box into smaller, cleaner sub-groups.

Stage 03

Measuring Messiness

To split effectively, the tree needs a way to measure how mixed a box is. We use a math rule called Gini Impurity.

A Gini score of 0.0 means the box is perfectly pure (all one fruit type). A score near 0.5 means it is completely mixed.

Gini = 1 - [ (P_apple)² + (P_orange)² ] Current Root P_apple = 12/26 ≈ 0.46 Current Root P_orange = 14/26 ≈ 0.54 Gini = 1 - (0.46² + 0.54²) ≈ 0.498 (Very Mixed)

💡 The tree’s ultimate objective is simple: Every time it makes a split, it wants to reduce the total combined Gini impurity of the child boxes as much as possible.

Stage 04

Testing Every Question

How does the tree find the best split? It tries testing candidate split points between observed values across the map.

Because the tree asks one feature question at a time (e.g., "Is Weight < 6?"), each possible split becomes a straight vertical or horizontal cut across the graph. Stage 05

The First Cut

The tree found the single best question! By splitting the data at this specific threshold, it gets the lowest possible combined Gini impurity.

This first split is often very influential because it shapes the rest of the tree. It divides the fruit map cleanly into two distinct regions.

Stage 06

Recursive Splitting

The tree isn't done. It now treats each of the two new boxes as a separate problem, and asks again: "Can I perform another split to make these even cleaner?"

It repeats this process recursively, carving the map into smaller and smaller axis-aligned rectangles until a box becomes pure, gets too small, or reaches the depth limit.

Stage 07

The Decision Boundary

Once the tree stops growing, we fill in the colors. Each rectangular box becomes a "Leaf Node". The model assigns a prediction to the entire box based on the fruit type that appears most often in that box.

Stage 08

Depth Limiting (Pre-Pruning)

Look closely at the tiny boxes the tree made just to capture one outlier fruit. The tree has started memorizing tiny details in the training dataset. This is called Overfitting.

If we give the tree new fruit, these hyper-specific rules will likely fail. We fix this by limiting the maximum depth of the tree (Pre-Pruning), so it learns general rules instead of tracking every detail.

Stage 09

Classifying Test Points

Now we bring in a small test dataset the model has not seen during training. We drop each test fruit (plotted as diamonds) onto the map. (Green border = correct prediction, red border = incorrect prediction).

At prediction time, the tree simply follows the learned rules from one split to the next until it reaches a leaf box.
