Stage 01

The Raw Data

Every machine learning model starts with data. Here we have measurements of fruits — each one scored on Sweetness and Color.

Each fruit is already labeled: it's either an Apple or an Orange. Because we have labels, this is supervised learning.

Training Dataset

💡 LDA needs labeled examples to learn the pattern that separates classes. Without labels, it cannot learn.

Stage 02

Choosing Good Features

Not every measurement helps separate classes. We pick features where the groups actually differ — like Sweetness and Color Score.

Bad features (random IDs, row numbers) add noise without signal. LDA works best when features genuinely explain class differences.

Feature Quality

✅ Sweetness — Apples tend to be sweeter
✅ Color Score — Apples score higher on color
❌ Row number — meaningless for classification
❌ Day of entry — no predictive power

💡 The plot highlights valid features. Notice how the two classes already form distinct clusters — a good sign for LDA.

Stage 03

Train / Test Split

We hold out a small test set (2 apples + 2 oranges) that the model will not see during training. The remaining 20 points are used to learn the pattern.

The test points are completely hidden until the evaluation stage. A model that only memorizes training data isn't actually smart — it's just a parrot with ambitions.

Stage 04

Group by Class

LDA separates the training data by label. All apples together, all oranges together. Now it can study each group independently.

These hulls give a simple visual outline of where each class's training points lie.

Stage 05

Compute Class Means

The mean is the center of each class — its average position in feature space. LDA uses these to understand where each group lives.

💡 The ★ markers on the plot show each class center. The gap between them is what LDA will try to exploit.

Stage 06

Within-Class Scatter

Not all apples are identical. SW measures how much the points in each class wiggle around their own mean.

The covariance ellipses on the plot show the spread shape. Diagonal entries = how much each feature varies. Off-diagonal = how features vary together.

Stage 07

The Shared Covariance Assumption

Here's LDA's key assumption: all classes share the same covariance shape. The groups can have different centers, but their spread is assumed to be roughly the same.

So LDA pools the individual covariance matrices into one shared estimate. Watch the ellipses morph into a single shape.

⚠️ If this assumption is badly wrong (classes have very different shapes), LDA can struggle. That's when Quadratic Discriminant Analysis (QDA) might work better.

Stage 08

Between-Class Scatter

SB measures how far apart the class centers are. The further apart, the easier it is for LDA to separate them.

LDA's goal is to maximize the ratio of between-class scatter to within-class scatter:

J(w) = wᵀ S_B w / wᵀ S_W w "I want the gap between group centers to be BIG, and the messiness inside each group to be SMALL."
Stage 09

The Optimal Direction

In this 2-class example, LDA gives one main discriminant direction w — the angle where classes appear most separated.

w ∝ S_W⁻¹ (μ₁ − μ₂) This isn't just the line between means. It also accounts for the spread inside each class.

Each point is projected onto this direction to get a 1D score. The projected score strip below the plot shows how well the classes separate along this axis.

Stage 10

The Decision Boundary

Because LDA is linear, the boundary between classes is a straight line (in 2D). Everything on one side is classified as Apple, the other as Orange.

The boundary comes from where the discriminant scores for both classes are equal:

δ_k(x) = xᵀ Σ⁻¹ μ_k − ½ μ_kᵀ Σ⁻¹ μ_k + log(π_k) Predict class with highest δ_k(x)
Stage 11

Classifying Test Points

Now the moment of truth. We take the test points the model has never seen and run them through the decision function.

Watch each test point light up with its prediction. Green rings = correct. Red rings = wrong.
