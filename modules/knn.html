Stage 01

The Raw Data

Every machine learning model starts with data. Here we have measurements of fruits — each one scored on Sweetness and Color Intensity.

Each fruit is already labeled: it's either an Apple or an Orange. Because we have labels, this is supervised learning.

Training Dataset

💡 KNN needs labeled examples to learn from. It doesn't build a formula — it simply remembers all the data and compares new points to what it already knows.

Stage 02

Choosing Good Features

Not every measurement helps separate classes. We pick features where the groups actually differ — like Sweetness and Color Intensity.

Bad features (random IDs, row numbers) add noise without signal. KNN is especially sensitive to irrelevant features because they pollute the distance calculation.

Feature Quality

✅ Sweetness — Oranges tend to be sweeter
✅ Color Intensity — Apples score higher on red
❌ Row number — meaningless for prediction
❌ Day picked — no predictive power

💡 Notice how the two classes form somewhat distinct clusters on the plot — a good sign. KNN relies entirely on spatial proximity.

Stage 03

Train / Test Split

We hold out a small test set (2 apples + 2 oranges) that the model will not see during training. The remaining points are used as the "memory" KNN will search through.

Testing on training data is one of the most common mistakes — it gives inflated confidence. A model that memorizes isn't smart; it's just a parrot.

Stage 04

Feature Scaling

This is the most common mistake with KNN. If Sweetness ranges 0–10 but Color ranges 0–100, the Color feature would completely dominate distance calculations.

We use Min-Max Scaling to squish every feature into the 0–1 range, giving each feature equal voice.

Scaled Value = (Value − Min) / (Max − Min) Example: Sweetness 7, Min=2, Max=9 → (7 − 2) / (9 − 2) = 5/7 ≈ 0.71

💡 Watch the points shift on the plot — they're being rescaled to [0, 1] on each axis. The shape of the clusters stays the same, but the axes are now comparable.

Stage 05

Measuring Distance

KNN's entire logic boils down to one question: "Who lives closest to me?" To answer that, we need a way to measure distance between points.

The most common choice is Euclidean distance — the straight-line distance between two points.

d(a, b) = √[(x₂ − x₁)² + (y₂ − y₁)²] This is just the Pythagorean theorem — the same formula you learned in geometry class.

Live Distance Calculation

Lines on the plot show the distance from a query point (★) to every training point. The darker lines emphasize the current teaching K nearest neighbors.

Stage 06

Choosing K

K is one of the most important hyperparameters in KNN — the number of neighbors to consult before making a prediction.

Small K (e.g. K=1): Very sensitive to noise. One outlier can flip the prediction.
Large K (e.g. K=15): Too smooth. Local patterns get drowned out by distant voters.

Adjust K

K = 3

K=3 is a balanced starting point. Try different values!

💡 Best practice: use an odd number for K to avoid ties. Use cross-validation to find the optimal K for your specific dataset.

Stage 07

Finding Neighbors

For our query point (★), we calculate the distance to every single training point, then sort from closest to farthest.

The top K points become the "neighborhood council" that will decide the prediction. Watch the circles expand outward from the query to find them.

Stage 08

The Majority Vote

Classification is now a democracy. Each of the K nearest neighbors gets one vote. The class with the most votes wins.

Vote Tally

If K=3 and neighbors are: 🍎 🍊 🍎 → Apple: 2 votes | Orange: 1 vote → Prediction: 🍎 Apple!

💡 For regression tasks, instead of voting, KNN takes the average value of the K neighbors. But for classification like ours, it's pure democracy.

Stage 09

Decision Boundary

What if we classified every possible point on the plot? The result is a colorful map showing which regions belong to which class.

Unlike linear models, KNN boundaries are non-linear — they bend and curve around the data naturally. The boundary is just wherever the majority vote flips from one class to the other.

Boundary Properties

The decision boundary emerges organically from the data — no formula is fitted. This is what makes KNN a non-parametric model.

💡 Small K = wiggly, complex boundaries. Large K = smoother boundaries. This is the classic bias-variance tradeoff.
